# Assignment Completion Checklist

Use this checklist to ensure all requirements are met before submission.

## Pre-Engagement Checklist

### System Selection
- [ ] Researched at least 5-6 penetration testing platforms
- [ ] Selected 4 systems/networks for testing
- [ ] Verified at least 3 systems released in past 3 months
- [ ] Confirmed systems complement each other (different OS, services, vulnerabilities)
- [ ] Verified systems enable complete network penetration test
- [ ] Confirmed no lab systems used (Metasploitable, DVWA, etc.)
- [ ] Created network diagrams for each system
- [ ] Documented selection justification

### Environment Setup
- [ ] Kali Linux VM installed and configured
- [ ] VPN access configured (if needed for platforms)
- [ ] All required tools installed and tested
- [ ] Documentation structure created
- [ ] Note-taking system set up
- [ ] Backup system in place

## Penetration Testing Checklist

### For Each System (1-4)

#### Intelligence Gathering
- [ ] Passive reconnaissance completed
- [ ] OSINT gathered
- [ ] DNS enumeration performed
- [ ] Subdomain discovery completed
- [ ] All findings documented
- [ ] Screenshots taken

#### Vulnerability Analysis
- [ ] Port scanning completed (Nmap)
- [ ] Service enumeration completed
- [ ] Version detection completed
- [ ] Vulnerability scanning completed (automated and/or manual)
- [ ] CVE research completed
- [ ] All vulnerabilities documented
- [ ] CVSS scores calculated

#### Exploitation
- [ ] Initial access achieved
- [ ] Proof of access documented (screenshots, flags)
- [ ] Exploitation steps documented
- [ ] Commands and outputs saved
- [ ] User flag obtained (if applicable)

#### Post-Exploitation
- [ ] System information gathered
- [ ] Privilege escalation attempted/achieved
- [ ] Root/Admin flag obtained (if applicable)
- [ ] Credentials extracted (if applicable)
- [ ] Lateral movement documented (if applicable)
- [ ] All activities documented
- [ ] Screenshots taken at each step

#### Documentation
- [ ] Complete walkthrough documented
- [ ] All commands saved
- [ ] All outputs saved
- [ ] Screenshots organized and labeled
- [ ] Appendix entry created

## Report Checklist

### Structure
- [ ] Title page
- [ ] Document control page
- [ ] Version control
- [ ] Table of contents
- [ ] List of figures
- [ ] List of tables

### Sections
- [ ] Executive Summary (2-3 pages)
  - [ ] Scope and objectives
  - [ ] Systems overview table
  - [ ] Critical analysis of difficulty
  - [ ] Summary of findings
  - [ ] Summary of recommendations

- [ ] Selection of Networks/Systems (3-4 pages)
  - [ ] Platform research and comparison
  - [ ] System selection justification
  - [ ] How systems complement each other
  - [ ] Network diagrams (one per system)

- [ ] Methodology (3-4 pages)
  - [ ] PTES framework overview
  - [ ] Detailed methodology steps
  - [ ] Risk rating methodology (CVSS)
  - [ ] References included

- [ ] Tools (2-3 pages)
  - [ ] Tool matrix table
  - [ ] Tool descriptions
  - [ ] Tool justifications
  - [ ] How tools complement each other

- [ ] Findings (8-10 pages)
  - [ ] Findings summary table
  - [ ] Detailed findings (all vulnerabilities)
  - [ ] CVSS scores for each
  - [ ] Impact analysis
  - [ ] Recommendations for each
  - [ ] References included

- [ ] Conclusions (2-3 pages)
  - [ ] Overall summary
  - [ ] Key themes
  - [ ] Limitations
  - [ ] Implications

- [ ] Reflection and Individual Contribution (2-3 pages)
  - [ ] Personal reflection (all 3 questions answered)
  - [ ] Group work reflection
  - [ ] Future improvements
  - [ ] Individual contributions (all team members)

- [ ] References
  - [ ] IEEE format
  - [ ] Alphabetical order
  - [ ] All in-text citations have references
  - [ ] All references have in-text citations

- [ ] Appendices
  - [ ] Appendix A: System 1 walkthrough
  - [ ] Appendix B: System 2 walkthrough
  - [ ] Appendix C: System 3 walkthrough
  - [ ] Appendix D: System 4 walkthrough
  - [ ] Each appendix detailed enough for reproduction

### Quality Checks
- [ ] Report length ~25 pages (main content)
- [ ] Professional formatting throughout
- [ ] Consistent font and style
- [ ] All figures have captions
- [ ] All tables have captions
- [ ] Page numbers correct
- [ ] Table of contents accurate
- [ ] No spelling/grammar errors
- [ ] Consistent terminology
- [ ] All screenshots clear and readable
- [ ] Proper section numbering

### Citation and Referencing
- [ ] IEEE citation style used
- [ ] All sources cited
- [ ] Walkthroughs/tutorials properly cited
- [ ] Figures/tables from sources cited
- [ ] Reference list complete
- [ ] No uncited content

## Academic Integrity Checklist

- [ ] No plagiarism
- [ ] All sources properly cited
- [ ] No content from past students
- [ ] No sharing with external parties
- [ ] Turnitin similarity checked (if available)
- [ ] Original work confirmed
- [ ] All team members reviewed for integrity

## Final Review

- [ ] Peer review completed by all members
- [ ] Report read by all team members
- [ ] Feedback incorporated
- [ ] Final proofreading completed
- [ ] All requirements met
- [ ] Ready for submission

## Common Issues to Avoid

- [ ] Using lab systems (Metasploitable, DVWA)
- [ ] Systems not diverse enough
- [ ] Insufficient documentation
- [ ] Missing references
- [ ] Poor formatting
- [ ] Incomplete appendices
- [ ] Missing screenshots
- [ ] Incorrect citation style
- [ ] Systems not recent enough
- [ ] Weak justifications
